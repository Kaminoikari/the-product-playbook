# 📎 ファイル統合ガイド

> ユーザーが補足資料をアップロードした時に読み込みます。

## 直接サポートされるファイルタイプ

| ファイルタイプ | 一般的なシナリオ | 統合方法 |
|-----------|-----------------|-------------------|
| **画像** | 競合スクリーンショット、ホワイトボード写真、手描きジャーニーマップ、アプリインターフェーススクリーンショット、データレポートスクリーンショット | コンテンツを認識し対応するステップに統合 |
| **PDF** | 市場レポート、社内ドキュメント、ユーザーリサーチレポート、既存PRD | 主要情報を抽出して統合 |
| **CSV / Excel** | ユーザー行動データ、リテンションデータ、NPS調査結果、営業データ | データを分析し定量評価に使用 |
| **テキストファイル** | インタビュー記録、既存要件ドキュメント、会議メモ | ペルソナ手がかり、ペインポイント、JTBDエビデンスを抽出 |
| **DOCX** | 既存PRD、プロダクトスペックドキュメント、ユーザーリサーチレポート | 主要情報を抽出して統合 |

## 直接サポートされないがガイダンスあり

| ファイルタイプ | ガイダンス |
|-----------|----------|
| **動画** | ユーザーにキーシーンの説明を依頼、またはトランスクリプト/字幕ファイルの提供を依頼 |
| **Figma / Sketch** | ユーザーにスクリーンショットを撮影してアップロードを依頼 |
| **Protobuf / Swagger** | ユーザーにJSONまたはテキストフォーマットでのエクスポートを依頼 |

## ファイル → ステップ自動マッピング表

| アップロード内容 | 自動マッピング先ステップ | 統合アクション |
|-----------------|-----------------|-------------------|
| 競合アプリ / ウェブサイトスクリーンショット | ポジショニング | 「競合代替品」として特定し、差別化を分析 |
| ホワイトボード手描きフローチャート | ジャーニーマップ / OST | フローを認識し構造化テーブルに変換 |
| ユーザーインタビュー記録 | ペルソナ + JTBD | ペインポイント、現在のワークアラウンド、感情的反応、Jobステートメントを抽出 |
| ユーザー行動データ（CSV） | 機会評価 + North Star + PMF | 仮定ベースのスコアを実データに置き換え |
| NPS / 満足度調査 | PMF + Sean Ellis Score | Scoreを直接計算；PMFレベルを決定 |
| 既存PRD / 要件ドキュメント | リビジョンモード S1 + MVP | 既存機能リストと決定履歴を抽出 |
| 市場レポートPDF | 機会評価 + 戦略 | 市場規模、トレンド、競合状況を抽出 |
| 営業データ | ビジネスモデル + GTM | 収益構造、顧客分布、チャネル効果を分析 |
| アプリインターフェーススクリーンショット | Aha Moment + ジャーニー | 現在の体験パスを分析 |

## 統合ルール

1. **積極的な特定**：まず「[ファイルタイプ]がアップロードされたことを確認しました。[主要情報]を[ステップ名]に統合します。」と説明
2. **既存出力を上書きしない**：アップロードされたコンテンツを「補足資料」としてマークし、出力を更新して変更伝播ルールをトリガー
3. **ソースをマーク**：出力でどのコンテンツがアップロードファイルから来たか注記（例：「✦ アップロードされたユーザーリサーチレポートから」）
4. **データ優先**：実データと以前の仮定がコンフリクトする場合、実データを優先
5. **ステップ横断の影響**：1つのファイルが複数のステップに影響する可能性がある — 影響を受けるすべてのステップを一度にリスト
